# LitReviewFlow 使用文档

## 1. 功能

LitReviewFlow 用于根据关键词或研究方向自动检索论文，并返回可供 AI 继续处理的结构化结果：

- 论文标题、作者、年份、期刊/会议
- DOI、OpenAlex/Semantic Scholar 链接
- 引用数、领域标签、开放 PDF
- 完整正文摘要字段 `abstract`
- 英文原始摘要别名字段 `abstract_en_raw`，用于区分“检索源返回/还原的英文摘要”和后续 AI 翻译摘要
- 面向 AI 的简洁摘要字段 `summary`

当前数据源：

- OpenAlex：覆盖广，支持语义检索，摘要需要从 `abstract_inverted_index` 自动还原。
- Semantic Scholar：AI/CS 相关性好，常直接返回纯文本摘要。

## 2. 安装

在项目根目录执行：

```powershell
python -m pip install -r requirements.txt
```

## 3. 配置

主配置文件是：

```text
config/config.json
```

首次使用可以复制模板：

```powershell
Copy-Item config/config.example.json config/config.json
```

然后在 `config/config.json` 中填写自己的 API Key。该文件用于本地运行，包含密钥时不要提交到 Git；公开仓库只提交 `config/config.example.json`。

可选配置项：

- `contact_email`：用于 OpenAlex User-Agent 的联系邮箱。
- `openalex_api_key`：OpenAlex API Key，可留空使用公开限额。
- `semantic_scholar_api_key`：Semantic Scholar API Key，没有也可以使用共享限额。
- `host` / `port`：本地 API 服务地址。
- `request_timeout_seconds`：单次请求超时时间。
- `request_delay_seconds`：预留的请求间隔参数。
- `max_retries`：外部 API 请求失败后的最大重试次数。
- `search_defaults.limit`：默认返回论文数量。
- `search_defaults.providers`：默认数据源，可选 `openalex`、`semantic_scholar`。
- `search_defaults.year_from` / `year_to`：默认年份范围。
- `search_defaults.sort`：默认排序，可选 `relevance`、`citation_count`、`year`。
- `search_defaults.require_abstract`：是否只返回有摘要的论文。
- `search_defaults.semantic_search`：OpenAlex 是否使用语义检索。
- `search_defaults.include_raw`：是否返回外部 API 原始数据。

## 4. 命令行使用

最常用：

```powershell
python scripts/search_literature.py "field-oriented control FOC motor drive PMSM" --limit 10
```

检索最新进展：

```powershell
python scripts/search_literature.py "field-oriented control FOC latest advances motor drive" --limit 10 --year-from 2024 --year-to 2026 --semantic-search
```

按引用量排序：

```powershell
python scripts/search_literature.py "large language model agents survey" --limit 10 --sort citation_count
```

按 DOI 精确回查论文元数据和英文摘要：

```powershell
python scripts/search_literature.py --doi "10.1117/1.2399537" --providers openalex --output english
```

将查询字符串当作 DOI 精确回查，而不是普通关键词检索：

```powershell
python scripts/search_literature.py "10.1117/1.2399537" --exact-doi --providers openalex --output english
```

批量 DOI 回查：

```powershell
python scripts/search_literature.py --doi-file .\dois.txt --providers openalex --limit 50 --output bilingual
```

经验规则：不要把 DOI 直接作为普通关键词搜索，否则搜索 API 可能返回相似但不相同的论文。需要核对英文摘要时，应使用 `--doi`、`--doi-file` 或 `--exact-doi`。

只使用 OpenAlex：

```powershell
python scripts/search_literature.py "PMSM field-oriented control" --providers openalex --limit 10
```

只使用 Semantic Scholar：

```powershell
python scripts/search_literature.py "PMSM field-oriented control" --providers semantic_scholar --limit 10
```

输出是 JSON，核心字段如下：

```json
{
  "query": "PMSM field-oriented control",
  "total": 10,
  "papers": [
    {
      "title": "...",
      "abstract": "...",
      "abstract_en_raw": "...",
      "abstract_zh": null,
      "summary": "...",
      "authors": ["..."],
      "year": 2026,
      "doi": "10.xxxx/xxxx",
      "url": "https://openalex.org/...",
      "source": "openalex",
      "citation_count": 12,
      "open_access_pdf": "https://..."
    }
  ]
}
```

## 5. 启动 API 服务

```powershell
python -m uvicorn litreviewflow.api:app --host 127.0.0.1 --port 8000
```

健康检查：

```text
GET http://127.0.0.1:8000/health
```

OpenAPI 描述：

```text
GET http://127.0.0.1:8000/openapi.json
```

AI 工具发现入口：

```text
GET http://127.0.0.1:8000/ai/tools
```

## 6. HTTP 接口

### 6.1 AI 简洁检索接口

推荐给其他 AI 调用：

```text
GET /ai/literature/search
```

示例：

```text
http://127.0.0.1:8000/ai/literature/search?query=field-oriented%20control%20FOC%20PMSM&limit=10&year_from=2024&year_to=2026&semantic_search=true
```

返回字段：

- `query`
- `total`
- `papers[].title`
- `papers[].year`
- `papers[].authors`
- `papers[].doi`
- `papers[].url`
- `papers[].source`
- `papers[].citation_count`
- `papers[].abstract`
- `papers[].abstract_en_raw`
- `papers[].summary`
- `papers[].open_access_pdf`

### 6.2 DOI 精确回查接口

推荐用于补齐已知 DOI 的英文摘要：

```text
GET /ai/literature/doi?doi=10.1117/1.2399537&providers=openalex
```

可重复传入多个 DOI：

```text
GET /ai/literature/doi?doi=10.1117/1.2399537&doi=10.1364/OE.19.019384&providers=openalex
```

### 6.3 完整 POST 检索接口

```text
POST /papers/search
```

示例 PowerShell：

```powershell
$body = @{
  query = "field-oriented control FOC PMSM motor drive"
  limit = 10
  providers = @("openalex", "semantic_scholar")
  dois = @()
  exact_doi = $false
  year_from = 2024
  year_to = 2026
  sort = "relevance"
  require_abstract = $true
  semantic_search = $true
  include_raw = $false
} | ConvertTo-Json

Invoke-RestMethod "http://127.0.0.1:8000/papers/search" -Method Post -ContentType "application/json" -Body $body
```

## 7. 给其他 AI 使用

其他 AI 可以优先读取：

```text
http://127.0.0.1:8000/ai/tools
http://127.0.0.1:8000/openapi.json
```

如果 AI 不能访问本地 HTTP 服务，也可以直接运行：

```powershell
python E:\14.TOOLs\LitReviewFlow\scripts\search_literature.py "你的研究方向" --limit 10
```

## 8. Codex Skill

已创建个人技能 `litreviewflow-search`。之后你可以这样问 AI：

```text
使用 litreviewflow-search 检索 FOC 最新研究，给我 10 篇论文的 DOI 和完整摘要。
```

或：

```text
Use $litreviewflow-search to search recent papers about PMSM field-oriented control and return DOI plus abstracts.
```
